Programové vybavení (software) je podmínkou využití technického vybavení (hardware). V raných dobách počítačové éry byly programy vytvářeny vždy pro konkrétní hardware. S rozvojem sériové výroby počítačů vznikla potřeba systémových programů, které by zajišťovaly základní operace s hardwarem a poskytly prostředí pro běh aplikací.

Operační systém

Základním programovým vybavením počítačů jsou operační systémy (OS – Operating System). Jejich hlavní funkcí je řízení hardware.

Jádro operačního systému

Řízení hardware zajišťuje jádro systému, které obsahuje subsystémy pro správu:

  • procesoru
  • paměti
  • diskových médií
  • dalších zařízení

Jádro využívá ovladače zařízení (drivers), pomocí nichž zpřístupňuje technické vybavení běžícím procesům.

Aplikační a uživatelské rozhraní

Nedílnou součástí operačních systémů je aplikační rozhraní (API - Application Programming Interface), které poskytuje programátorům nástroje pro vývoj aplikací.

Uživatelské rozhraní slouží k ovládání OS a spouštění aplikací:

  • CLI příkazový řádek (starší systémy)
  • GUI grafické uživatelské rozhraní (moderní systémy)

Historie operačních systémů

Mezi první moderní operační systémy patří víceúlohový a víceuživatelský systém UNIX, používaný již od 70. let.

V 80. letech si konkurovaly zejména:

  • MS-DOS (Microsoft, IBM PC)
  • Mac OS (Apple, průkopník grafického rozhraní)

Na grafickém rozhraní byl později založen i systém MS Windows, který dominoval zejména na přelomu tisíciletí. Současně rostla popularita otevřeného systému Linux, vytvořeného Linusem Torvaldsem.

Rozdělení operačních systémů

Současné operační systémy rozdělujeme podle typu zařízení:

  • Serverové systémy: MS Windows Server, Linux, Unix
  • Osobní počítače: MS Windows, Linux Ubuntu, macOS
  • Mobilní zařízení: Android, iOS, Windows Phone, Symbian
  • RTOS (Real-Time Operating System): systémy reálného času pro řízení a měření

Systémy reálného času (RTOS)

RTOS zajišťují provoz „chytrých“ zařízení, například robotických kamer, měřicích přístrojů nebo průmyslových řídicích jednotek. Na rozdíl od běžných systémů musí RTOS pracovat bez prodlevy a s vysokou spolehlivostí v reálném čase.
